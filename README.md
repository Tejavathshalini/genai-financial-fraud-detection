## Workflow Architecture

![Workflow](images/workflow.png)

The system follows a structured fraud detection pipeline:

1. Transaction dataset loading
2. Data preprocessing and cleaning
3. Feature selection
4. StandardScaler normalization
5. Isolation Forest anomaly detection
6. Fraud classification
7. Interactive dashboard
8. GenAI fraud reasoning

## Machine Learning Model

The system uses **Isolation Forest**, an unsupervised anomaly detection algorithm designed to identify rare or unusual transaction patterns.

Transactions predicted as anomalies are labeled as **Suspicious**, while normal patterns are labeled as **Normal**.
